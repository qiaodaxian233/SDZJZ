# 一键建造结构（结构蓝图）设计

## 目标
攒够材料 → 右键 → 直接把一个预制的大型结构摆进世界（作者用 3D 建模 + ObjToSchematic 体素化）。

## 制作管线
1. 3D 模型(.glb/.obj) → **ObjToSchematic** 体素化。
2. 导出 **.nbt（Structure blocks 格式）**——原版/Fabric 可直接加载。
   - 单个 ≤ 48×48×48（原版结构方块上限）。更大：ObjToSchematic 会切成多块 .nbt，或把 Size 调小成单块。
3. .nbt 放 `data/sdzjz/structures/<name>.nbt`。
4. Fabric 用 `StructureTemplateManager.getTemplate(id)` 加载，`StructureTemplate.place(...)` 摆放。

## 玩法
- 「建造蓝图」物品（可每种结构一个蓝图，或一个通用蓝图选结构）。
- 右键地面：校验材料（身上背包 / 或连接的数据面板里）→ 足够则扣除 → 在指向位置摆出结构。
- 材料成本每结构单独定（类似配方）。可选：需要"图纸/研究"解锁。

## 更新：格式改用 .mcfunction（网页版无 .nbt）
ObjToSchematic 网页版可选：.litematic / .schem / .schematic(旧) / .mcfunction / Command Blocks / .ots_blocks / .json / .txt / .glb(PRO)。
- **采用 `.mcfunction`**：内容是 setblock 命令，Fabric 端逐行解析（坐标+方块，用 BlockArgumentParser 解析方块状态），相对 `~` 坐标就按目标点偏移摆放。不限尺寸、可读可核对。
- 放 `data/sdzjz/structures/<name>.mcfunction`（或作为资源随包）。
- 备选 .litematic：标准、单文件大结构，但需自写 gzip-NBT + 调色板 + 位打包解析（Litematica 位流跨 long 连续，易错），有需要再做。
- 流程不变：蓝图物品右键 → 校验/扣材料 → 解析 mcfunction → 在指向位置逐块摆出（大结构可分 tick 摆以防卡顿）。
