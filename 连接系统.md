# 连接系统（走线）设计 —— 参考 AE

## 现状（第一版，已实现）
- 结构核心产出时，从核心出发做 BFS：
  1. **相邻**（六面）有数据面板/箱子 → 直接存入；
  2. 遇到**数据线**方块则顺着继续路由，把产物送到线路末端连接的数据面板/箱子。
- 于是可以「核心 → 拉一串数据线 → 远处的数据面板/箱子」，不用非得贴着核心。
- 无电力、无通道限制（对齐本模组"去电力"设定），BFS 上限 256 格。

## 与 AE 的对照 / 后续路线
- **AE 网络** = 线缆连出一张网，存储/机器/终端都挂网上，物品走"逻辑网络"而非实体管道。我们的数据面板已是"逻辑存储"，很契合。
- 下一步可做的（按需）：
  1. **多目标分发**：核心把产物分发到网络里"能装下的第一个/多个"面板，而不是只找最近一个。
  2. **共享网络存储**：同一张数据线网络上的多个数据面板合并成一个逻辑库；任意终端可取全网物品（AE 的"ME 网络"感）。
  3. **无线/远程终端**：手持终端查看/取用整张网络。
  4. **细导线模型 + 连接贴图**：把整块数据线换成细导管，六面自动连接（AE 线缆观感）。
  5. **输入路由**：消耗类机器从网络里"取料"，而不只是往外送。
- 这些都不涉及电力/通道，只做连接与逻辑存储聚合。

## 备注
本模组核心是"量产机器 + 逻辑存储"，走线服务于"把产物送进存储/下一台机器"，不追求 AE 那套通道/信道复杂度。
