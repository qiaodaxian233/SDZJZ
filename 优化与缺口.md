# 优化与缺口盘点（m48）

> 如实盘点：哪些还要优化、哪些生电内容没做。带优先级，做一项划一项。

---

## 一、需要优化的（按收益排）

### 性能
- [x] **输出/取料路由缓存**（m48 已做）：pushOutput 和消耗机取料原来每个生产周期跑 4 连 BFS，现在目标坐标缓存 40 tick。
- [x] 面板刷新节流（m47）、路由不强制加载区块（m47）。
- [ ] **核心→客户端同步瘦身**：现在节点变动是整个 BE NBT 全量同步（含所有节点+连线+缓存），节点多时包很大。改增量同步或按需同步。
- [ ] **画布节点很多时的渲染**：目前每帧遍历全部节点画框+升级格+连线，几百节点会掉帧。加视口裁剪（只画屏内节点）。
- [ ] **内部缓存 NBT**：连线物流缓存跟着 BE 全量存/同步，种类多时膨胀。设上限已有（20 万），可再加种类上限。

### 体验（UX）
- [ ] **数据面板搜索只认英文 id**：搜"铁"搜不到 iron。因为过滤在服务端、翻译在客户端。方案：客户端把可见语言的匹配结果转成 id 集合发服务端，或改为客户端过滤架构。中文玩家刚需，优先级高。
- [ ] **机器 tooltip**：悬停机器物品显示"产什么/周期/是否吃料"。不用进画布就能看懂一台机器。
- [ ] **画布节点显示产率**：节点框上加"×N/秒"实时产率，一眼看出瓶颈。
- [ ] **JEI/REI 兼容**：超大工作台的 44 个大配方接进 REI，查配方不用翻文档。工程量中等。
- [ ] **配方浏览器搜索**：44 台了，右侧列表该加个搜索框。
- [ ] **数值全面 config 化**：机器产率/周期目前写死在 Machines.java，搬进 config 让玩家自己调。

### 健壮性
- [ ] **连线按边精确路由**：现在是粗粒度共享池（所有出线机器 → 一个池 → 所有入线机器），A→B、C→D 两条独立链会互相偷料。改为按边分池。
- [ ] **老存档升级兼容测试**：m43 存储重构后，旧存档里数据面板存的东西已不在（面板不再自带存储）。如实说：**旧面板里的东西升级后会消失**，需要提示或迁移逻辑。

## 二、还没做的生电内容（对照大清单）

### 机器类（还缺的）
- [x] 恶魂塔（m48）— 火药/恶魂泪
- [x] 旋风人塔（m48，1.21 试炼密室新怪）— 旋风棒/风弹
- [x] 骨粉机（m48）— 骨粉量产
- [x] 苔藓机（m48）— 苔藓块/苔藓地毯
- [x] 切石机（m48，消耗类）— 石头→石砖
- [ ] **酿造塔**（消耗类：下界疣+水瓶+材料→药水）— 药水是物品组件（Potion component），比普通消耗机复杂一档，单独做。
- [ ] **自动合成机**（消耗类：吃材料按原版配方产任意物品）— "量产一切"的最后一块拼图，需要接原版 RecipeManager。大件。
- [ ] **坚守者（Warden）农场** — 幽匿系列（幽匿催化剂/尖啸体）。原版 Warden 不掉幽匿方块（靠催化剂机制），设计上要按"幽匿蔓延收割"来做，不是普通刷怪掉落。
- [ ] 嘎吱怪（Creaking）— **1.21.2+ 才有，1.21.1 没有这个生物**，暂不做（版本升级后再说）。
- [ ] 试炼钥匙/不祥之瓶农场 — 试炼奖励不是掉落物，机制特殊，待设计。

### 系统类（大件，按优先级）
1. **村民系统**（你点名要独立 UI）：村民繁殖机 + 交易所 + 打折机。设计方向：村民作为"数据化村民"存进机器（类似抓物笼子），交易所 UI 列出职业交易表、吃料出货，打折机制对应治愈僵尸村民折扣。**下一个大件就做这个。**
2. **经验（XP）系统**：生电刷怪塔一半的价值是经验（修补/附魔）。方向：机器产出"经验瓶"或"存储经验"，配一个经验提取器。
3. **附魔体系**：附魔书量产（和村民图书管理员挂钩，依赖村民系统）。
4. **潜影盒装卸**：仓储侧的打盒/拆盒机（配合数据面板导出）。

### 范围外（F 桶，之前已定不做进机器系统）
世吞、珍珠炮、屠龙炮 — 红石工程装置，不是"插进核心的机器"，要做得单独立项。
